# Dochazka Generator Telegram Bot — Design Spec

## Problem

Друг получает от бухгалтера Excel-файл (Mzdy) с часами сотрудников кофейни по дням. Из него нужно вручную составить второй Excel — книгу посещаемости (Docházka) с конкретным временем прихода/ухода. Существующий Python-скрипт автоматизирует это, но запускать его на компьютере слишком сложно для нетехнического пользователя. Скрипт также ломался из-за отклонений в формате файла.

## Solution

Telegram-бот на Raspberry Pi. Пользователь отправляет файл Mzdy — получает Docházku в ответ. Старый скрипт удаляется, вся логика пишется с нуля внутри бота.

## Decisions

| Вопрос | Решение |
|--------|---------|
| Доступ | Открытый для всех (без whitelist) |
| Формат файла | Строгий, по шаблону. Ошибки — понятным текстом на русском |
| Команда /template | Не нужна, шаблон отправляется вручную один раз |
| Ответ бота | Файл `Dochazka_MM.YYYY.xlsx` + краткая сводка (сотрудники, часы) |
| Текст / не-xlsx | "Отправь мне файл Mzdy в формате .xlsx" |
| Имя файла | Требуется формат `Mzdy_MM.YYYY.xlsx`, при несовпадении — инструкция с примером |
| Старый скрипт | Удаляется, не поддерживается |

## Architecture

### File Structure

```
dochazka-generator/
├── bot.py                          # точка входа, aiogram хендлеры
├── generator.py                    # парсинг Excel, смены, генерация Docházka
├── .env                            # BOT_TOKEN
├── requirements.txt                # aiogram, openpyxl, python-dotenv
├── dochazka-generator.service      # systemd unit
└── docs/
```

### bot.py — Telegram-интерфейс

**Команды:**
- `/start` — приветствие: "Отправь мне файл Mzdy_MM.YYYY.xlsx и я сгенерирую книгу посещаемости (Docházku)."

**Хендлер документов:**
1. Проверить, что файл имеет расширение `.xlsx`
2. Проверить, что имя файла соответствует паттерну `Mzdy_MM.YYYY.xlsx` (регистронезависимо)
3. Если не совпадает — отправить сообщение: "Файл должен называться в формате Mzdy_MM.YYYY.xlsx (например, Mzdy_03.2026.xlsx). Переименуй файл и отправь снова."
4. Скачать файл во временную директорию (`tempfile.mkdtemp()`)
5. Вызвать `generator.generate(input_path, output_path)`
6. Отправить результат как документ + текстовую сводку
7. Удалить временные файлы (в `finally`)

**Хендлер текста и прочего:**
- Ответ: "Отправь мне файл Mzdy в формате .xlsx"

**Обработка ошибок:**
- `DochazkaError` — отправить `error.message` пользователю (понятный текст на русском)
- Любое другое исключение — "Произошла непредвиденная ошибка. Попробуй ещё раз или обратись к администратору."

### generator.py — бизнес-логика

**Классы и функции:**

```python
class DochazkaError(Exception):
    """Ошибка с понятным сообщением для пользователя."""
    pass

def generate(input_path: str, output_path: str) -> Summary:
    """Главная функция. Парсит Mzdy, генерирует Docházku.
    Возвращает Summary с данными для сводки.
    Бросает DochazkaError с понятным сообщением при ошибках."""
```

**Summary** — dataclass с полями:
- `month: int`
- `year: int`
- `employees: dict[str, float]` — {имя: всего часов}

**Парсинг Mzdy (из имени файла + содержимого):**
- Месяц/год — из имени файла (уже провалидировано ботом)
- Найти лист с данными — ищет "час" в ячейке B1
- Имена сотрудников — строка 1, столбцы D+
- Данные по дням — строки 5–35

**Ошибки парсинга (DochazkaError):**
- "Не нашёл лист с данными. В ячейке B1 первого листа должно быть слово 'часов'."
- "Не нашёл имена сотрудников в строке 1 (столбцы D и далее)."
- "Нет данных по дням (строки 5–35 пустые)."
- "Не удалось прочитать файл. Убедись, что это файл Excel (.xlsx)."

**Распределение смен:**
- Время открытия: будни 6:30, выходные 8:00
- Закрытие = открытие + часы работы кофейни (столбец B)
- 1 сотрудник: начинает с открытия
- 2 сотрудника: первый открывает, второй закрывает, перекрытие в середине
- 3+ сотрудников: последовательно друг за другом от открытия

**Генерация Docházka xlsx:**
- Отдельный лист на каждого активного сотрудника
- Заголовок: "Kniha docházky", месяц/год, полное имя
- Столбцы: Datum, Příchod, Odchod, Přestávka, Odpracováno, Podpis zaměstnance, Podpis zaměstnavatele, Poznámka
- Строка итогов: Součet hodin

### Deployment

- systemd user service `dochazka-generator.service`
- `Type=simple`, `Restart=on-failure`, `RestartSec=10`
- `EnvironmentFile` для `.env`
- Как у lamp-bot и telegram-digest

### Config (.env)

```
BOT_TOKEN=<telegram bot token>
```

Единственная настройка — токен бота. Время открытия (6:30/8:00) — константы в `generator.py`.
